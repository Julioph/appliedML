A time series prediction example: forecasting electricity load

We will see an example of how to use machine learning to make predictions in time series. The example we'll see is a time series that represents the consumption of electrical power in Texas over a few years. (This example was inspired by the following blog post.)

Using an autoregressive model for time series prediction

The first model that we will consider for time series prediction is called an autoregressive model or AR model. This is a linear model where the next entry in the time series is modeled as a linear function of the pp previous entries:

Ytc+i=1paiYtiYtc+i=1paiYti
We write a function that trains an AR model. It will go through the training time series and generate training instances. Each training instance will consist of the pp previous measurements, and the desired output value at that time point.

These instances are then used to train the time series prediction model. For a linear AR model, we can treat this as a linear regression problem, and we can use any of the linear regressors in scikit-learn.

In [7]:
def train_ar_model(Ytrain, p, base_model):
    train_input = []
    train_output = []
    for i in range(p, len(Ytrain)):
        train_input.append(Ytrain[i-p:i])
        train_output.append(Ytrain[i])
    return base_model.fit(train_input, train_output)

We then write a function to evaluate an autoregressive model. It will generate instances in the same way as we did for the training data, but now we'll use them for prediction instead. We then compute the MSE and R2R2 to evaluate. We also plot the observed values and the predicted values.

In [8]:
from sklearn.metrics import mean_squared_error, r2_score

def test_ar_model(model, Ytest, p):
    test_input = []
    for i in range(p, len(Ytest)):
        test_input.append(Ytest[i-p:i])
    guess = model.predict(test_input)
    mse = mean_squared_error(Ytest[p:], guess)
    r2 = r2_score(Ytest[p:], guess)
    print(f'MSE = {mse:.4f}, R = {r2:.4f}')
    
    plt.plot(Ytest[p:200])
    plt.plot(guess[:200-p])

Now, let's evaluate how well an AR(2)AR(2) model performs.

As you can see, it performs quite well. We get an R2R2 score of 0.99, which is very high. And as you can see in the curve, the predicted values are very close to the observed values.

But how useful is it to predict the load just half an hour into the future? Couldn't we try to predict even further into the future?

We write a function extrapolate_ar_model that will extrapolate in a step-by-step fashion. As in the named entity tagging example, we are now using features based on our own predictions in the previous steps.

This function is used for illustration. It will start at a number of different time points, and extrapolate a given number of steps into the future. It will then plot the extrapolated predictions.

As we can see, the linear autoregressive model is not that good at predicting the electricity load beyond 30 minutes: the extrapolations quickly diverge from the actual observed loads.

def extrapolate_ar_model(model, Ytest, p, steps, indices):
    for i in indices:
        previous = list(Ytest[i-p:i])
        x_g = []
        y_g = []
        for j in range(steps):
            guess = model.predict([previous])[0]
            y_g.append(guess)
            x_g.append(i+j)
            previous = previous[1:] + [guess]
        plt.plot(x_g, y_g, '.-')

extrapolate_ar_model(model, Ytest, p, 40, [30, 65, 145])

To measure quantitatively what we visualized above, the following function will extrapolate a given number of time steps into the future for every position in the time series. It will then compare the extrapolated predictions to the gold-standard values using MSE and R2R2.

We measure how well the linear autoregressive model can predict 10 steps (that is, 5 hours) into the future. As you can see, the linear model performs quite poorly here, getting an R2R2 of just 0.14.

def test_ar_model_extrapolated(model, Ytest, p, steps):
    test_outputs = []
    guesses = []
    for i in range(p+steps-1, len(Ytest)):
        previous = list(Ytest[i-p-steps+1:i-steps+1])
        for j in range(steps):
            guess = model.predict([previous])[0]
            previous = previous[1:] + [guess]
        guesses.append(guess)
        test_outputs.append(Ytest[i])
    mse = mean_squared_error(test_outputs, guesses)
    r2 = r2_score(test_outputs, guesses)
    print(f'MSE = {mse:.4f}, R = {r2:.4f}')
    
test_ar_model_extrapolated(model, Ytest, p, 10)

MSE = 1565019.9587, R = 0.1398

Using a nonlinear autoregressive model

Now, let's go beyond the linear AR model and try a nonlinear model (or NAR model). In this example, we will use a random forest regressor. It will base its predictions on the 20 previous measured values.

This gives a MSE value that is substantially lower than what we saw for the linear AR model.

How you can extend this work

This worked surprisingly well. So could we use this approach to build a stock market predictor and make a ton of money? Probably you could use this model as well as any other model, but it would probably be much less impressive in that case, since the time series we used here has an extremely predictable shape. Stock market data tends to be much more unpredictable and "noisy".

It can be noted that the current approach did not use any exogeneous variables. For instance, as we saw above, the dataset contains quite a bit of information, including variables such as humidity, electricity price, etc. Also, since there are very strong periodic tendencies here (seasonality), it would probably be very useful to include features that express the time of day and weekday. Time-based features are also quite nice, because we will know their future values! If we include this information, the linear model would probably also perform better for extrapolation.
