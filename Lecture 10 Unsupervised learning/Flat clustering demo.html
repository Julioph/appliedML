#Generate synthetic data
np.random.seed(0)
def make_data(N):
    X = []
    for _ in range(N):
        if np.random.random() < 0.3:
            X.append(0.7*np.random.normal(size=2))
        elif np.random.random() < 0.7:
            r = np.random.normal()+5
            a = np.pi * np.random.random() + np.pi/5
            X.append([r*np.cos(a), r*np.sin(a)])
        else:
            r = 0.7*np.random.normal()+7
            a = -np.pi/2 * np.random.random()
            X.append([r*np.cos(a), r*np.sin(a)])
            
    return np.array(X)

X = make_data(450) 

# make scatter plot of X, with each point represented by a filled-circle with edgecolor black ('k')
plt.scatter(X[:,0], X[:,1], edgecolors='k');

Choosing the number of clusters is not a trivial task. Sometime 'elbow' method is used to help determine the number of k to use. But it is not always reliable.

    inertia.append(np.sqrt(kmeans.inertia_))

Ideally, one should set the values for 'eps' and 'min_samples' parameters when using DBSCAN, eventhough default values are given. The eps should be set to the maximum distance between two data points to be considered in the same neighborhood. While the min_samples parameter should be set to the minimum amount of data points in a neighborhood to be considered a cluster.
